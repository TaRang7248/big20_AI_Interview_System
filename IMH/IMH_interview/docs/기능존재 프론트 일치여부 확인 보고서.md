# 기능존재 프론트 일치여부 확인 보고서 (정밀 검증 버전)

본 보고서는 백엔드 기능 명세(`docs/기능존재 확인 보고서.md`)의 36개 항목에 대하여 실제 프론트엔드 UI/UX 흐름, 데이터 거버넌스(Data Governance) 원칙 준수 여부, 그리고 정책 동결(Policy Freeze)의 강제성을 정밀 검증한 결과입니다.

---

## 데이터 거버넌스 정의 및 검증 기준

본 보고서에서는 공고 데이터를 다음 4가지 유형으로 분류하여 검증합니다.

| 분류 | 대상 항목 | LLM 전달 | 수정 가능 시점 | UI 강제 사항 |
|:---|:---|:---:|:---|:---|
| **공고 입력 정보** | 제목, 설명, 태그 | 일부 전달 | 게시 전 | 게시 후 수정 지양 |
| **AI 정책 정보** | 질문 수, 제한시간, 가중치 | **전달됨** | **게시 전** | **게시 후 수정 불가 (Freeze)** |
| **운영 Metadata** | 모집 인원, 마감일, 근무지 | 아님 | 상시 가능 | 제한 없음 |
| **시스템 무결성** | Job ID, 상태, 평가 해시 | 아님 | 불가 | 수정 UI 노출 금지 |

## 정밀 검증 결과 상세

### Q1 프론트엔드 상세 기능
- **Question**: 현재 프론트는 어떤 기능들이 있는지 상세히 설명해.
- **Backend Capability**: API 및 스키마 설계 완료 (Admin/Candidate 구분)
- **Frontend UI Presence**: 관리자(공고/통계), 지원자(이력서/면접/결과) 대시보드 완비
- **Frontend UX Flow**: 각 역할별 엔드포인트 연동 및 페이지 네비게이션 정상 작동
- **Data Governance / Policy Consistency**: PrivateRoute 및 토큰 기반 권한 분리 적용됨
- **Evidence**: `frontend/src/App.jsx` (Route mapping), `frontend/src/services/api.js`
- **Conclusion**: Frontend/Backend Fully Consistent

### Q2 답변 평가 기준

- **Frontend UI Presence**: 관리자 상세 페이지 레이더 차트 및 역량별 점수 UI
- **Frontend UX Flow**: 세션 종료 후 서버에서 계산된 정량 점수를 실시간 바인딩하여 표시
- **Data Governance / Policy Consistency**: 지원자용 결과 뷰에서는 상세 점수 및 해시 데이터가 DOM에서 원천 제거됨 (Admin Boundary 준수)

- **Data Governance / Policy Consistency**: 핵심 AI 정책 정보임에도 불구하고 관리적 제어권이 UI에 위임되지 않음

### Q5 공고 정책 동결 및 수정 제한
- **Question**: 공고 정책 동결(Job Policy Freeze) 및 데이터 거버넌스 설계원칙
- **Backend Capability**: 게시(Publish) 시 Snapshot 생성 및 수정 시도 시 403/409 에러 반환
- **Frontend UI Presence**: 수정 페이지(`AdminPostingEdit.jsx`) 존재함
- **Frontend UX Flow**: **정상 흐름 실패**. 게시된 공고임에도 불구하고 UI에서 정책 필드가 잠기지 않아 사용자가 수정을 시도할 수 있으며, 실제 제출 후 서버 에러를 통해서만 인지 가능
- **Data Governance / Policy Consistency**: **Data Governance Violation**. AI 정책 정보(질문 수 등)는 게시 후 UI에서 즉시 `disabled` 처리되어야 하나, 시각적/기능적 잠금이 구현되지 않음
- **Evidence**: `AdminPostingEdit.jsx` (L13-28: status 기반 disabled 처리 로직 부재)
- **Conclusion**: Data Governance Violation
